# Code Reviewer Skill

## Instructions

When reviewing Java code:
1. **Java 25 Best Practices**: Check for proper use of pattern matching, virtual threads, switch expressions, and structured concurrency.
2. **Spring Conventions**: Verify proper usage of annotations like `@RestController`, `@Service`, `@Repository`, and check constructor injection.
3. **Security Analysis**: Identify potential vulnerabilities (SQL injection, unsafe input handling, lack of validation).
4. **Code Quality**: Identify logic bugs, potential null pointer exceptions, and suggest refactoring for readability.
5. **Actionable Suggestions**: Provide concrete code examples for any suggested improvements.
